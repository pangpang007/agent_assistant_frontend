## 1. 目标

实现工作流执行引擎的前端部分，使用户能够：

1. 在编辑器中一键触发工作流运行
2. 通过 WebSocket 实时接收每个节点的执行状态变更与日志
3. 在右侧执行面板中查看执行进度、节点状态列表、实时日志流、统计数据
4. 画布上的节点同步显示执行状态（颜色变化 + 动画）
5. 在审核节点暂停时进行人工干预（通过 / 驳回 / 修改后通过）
6. 执行完成后查看汇总结果，执行失败时查看错误信息并重试
7. 随时停止执行

### 1.1 功能范围总览

| 模块 | 说明 |
|------|------|
| **A. 运行入口** | 顶部工具栏「运行」按钮 + 运行配置弹窗 |
| **B. WebSocket 连接管理** | `useWebSocket` hook、消息协议、自动重连、心跳 |
| **C. 实时执行面板** | 运行时替换右侧属性面板，展示进度/状态/日志/统计 |
| **D. 审核节点交互** | 审核内容展示、通过/驳回/修改后通过、审核记录 |
| **E. 执行完成/失败** | 最终输出汇总、错误详情、重试、关闭面板 |
| **F. 画布执行状态同步** | 节点边框颜色 + 动画 + 连线高亮 |
| **G. 执行中断** | 停止执行按钮 + 确认弹窗 |

```tsx
// RightPanelContainer.tsx 更新
const RightPanelContainer = () => {

  const isExecuting = useExecutionStore((s) => s.status !== 'idle');

    return <ExecutionPanel />;
```

## 3. WebSocket 连接管理

### 3.1 useWebSocket Hook 设计

**文件路径**：`src/hooks/useExecutionWebSocket.ts`

```ts
import { useCallback, useEffect, useRef, useState } from 'react';

// WebSocket 连接状态
type WSConnectionStatus = 'disconnected' | 'connecting' | 'connected' | 'reconnecting';

interface UseExecutionWebSocketOptions {
  executionId: string | null;
  onMessage: (message: WebSocketMessage) => void;
  onStatusChange?: (status: WSConnectionStatus) => void;
  autoConnect?: boolean;
}

interface UseExecutionWebSocketReturn {
  status: WSConnectionStatus;
  send: (message: WebSocketMessage) => void;
  connect: () => void;
  disconnect: () => void;
  lastMessage: WebSocketMessage | null;
}

function useExecutionWebSocket({
  executionId,
  onMessage,
  onStatusChange,
  autoConnect = false,
}: UseExecutionWebSocketOptions): UseExecutionWebSocketReturn {
  const wsRef = useRef<WebSocket | null>(null);
  const heartbeatTimerRef = useRef<NodeJS.Timeout | null>(null);
  const reconnectTimerRef = useRef<NodeJS.Timeout | null>(null);
  const reconnectCountRef = useRef(0);
  const [status, setStatus] = useState<WSConnectionStatus>('disconnected');
  const [lastMessage, setLastMessage] = useState<WebSocketMessage | null>(null);

  // 最大重连次数
  const MAX_RECONNECT_ATTEMPTS = 5;
  // 重连间隔基数（毫秒）
  const RECONNECT_BASE_DELAY = 1000;
  // 心跳间隔（毫秒）
  const HEARTBEAT_INTERVAL = 30000;

  const updateStatus = useCallback((newStatus: WSConnectionStatus) => {
    setStatus(newStatus);
    onStatusChange?.(newStatus);
  }, [onStatusChange]);

  // 构建 WebSocket URL
  const buildWsUrl = useCallback(() => {
    const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:';
    const host = window.location.host;
    return `${protocol}//${host}/api/executions/${executionId}/stream`;
  }, [executionId]);

  // 启动心跳
  const startHeartbeat = useCallback(() => {
    stopHeartbeat();
    heartbeatTimerRef.current = setInterval(() => {
      if (wsRef.current?.readyState === WebSocket.OPEN) {
        wsRef.current.send(JSON.stringify({ type: 'ping' }));
      }
    }, HEARTBEAT_INTERVAL);
  }, []);

  const stopHeartbeat = useCallback(() => {
    if (heartbeatTimerRef.current) {
      clearInterval(heartbeatTimerRef.current);
      heartbeatTimerRef.current = null;
    }
  }, []);

  // 重连策略
  const scheduleReconnect = useCallback(() => {
    if (reconnectCountRef.current >= MAX_RECONNECT_ATTEMPTS) {
      updateStatus('disconnected');
      return;
    }

    updateStatus('reconnecting');
    // 指数退避：1s → 2s → 4s → 8s → 16s
    const delay = RECONNECT_BASE_DELAY * Math.pow(2, reconnectCountRef.current);
    reconnectCountRef.current += 1;

    reconnectTimerRef.current = setTimeout(() => {
      connect();
    }, delay);
  }, []);

  // 连接
  const connect = useCallback(() => {
    if (!executionId) return;

    // 关闭已有连接
    if (wsRef.current) {
      wsRef.current.close(1000, 'Reconnecting');
    }

    updateStatus('connecting');
    const ws = new WebSocket(buildWsUrl());
    wsRef.current = ws;

    ws.onopen = () => {
      updateStatus('connected');
      reconnectCountRef.current = 0;
      startHeartbeat();
    };

    ws.onmessage = (event) => {
      try {
        const message: WebSocketMessage = JSON.parse(event.data);
        // 忽略 pong 消息
        if (message.type === 'pong') return;
        setLastMessage(message);
        onMessage(message);
      } catch (err) {
        console.error('[WS] Failed to parse message:', err);
      }
    };

    ws.onclose = (event) => {
      stopHeartbeat();
      // 非正常关闭 → 尝试重连
      if (event.code !== 1000) {
        scheduleReconnect();
      } else {
        updateStatus('disconnected');
      }
    };

    ws.onerror = (error) => {
      console.error('[WS] Error:', error);
    };
  }, [executionId, buildWsUrl, onMessage, startHeartbeat, stopHeartbeat, scheduleReconnect, updateStatus]);

  // 断开
  const disconnect = useCallback(() => {
    stopHeartbeat();
    if (reconnectTimerRef.current) {
      clearTimeout(reconnectTimerRef.current);
      reconnectTimerRef.current = null;
    }
    reconnectCountRef.current = MAX_RECONNECT_ATTEMPTS; // 阻止重连
    if (wsRef.current) {
      wsRef.current.close(1000, 'User disconnect');
      wsRef.current = null;
    }
    updateStatus('disconnected');
  }, [stopHeartbeat, updateStatus]);

  // 发送消息
  const send = useCallback((message: WebSocketMessage) => {
    if (wsRef.current?.readyState === WebSocket.OPEN) {
      wsRef.current.send(JSON.stringify(message));
    }
  }, []);

  // 自动连接
  useEffect(() => {
    if (autoConnect && executionId) {
      connect();
    }
    return () => {
      disconnect();
    };
  }, [autoConnect, executionId]);

  return { status, send, connect, disconnect, lastMessage };
}
```

### 3.3 重连策略

| 参数 | 值 | 说明 |
|------|-----|------|
| 最大重连次数 | 5 | 超过后标记为 `disconnected` |
| 退避策略 | 指数退避 | `1s → 2s → 4s → 8s → 16s` |
| 重连条件 | `event.code !== 1000` | 正常关闭（code=1000）不重连 |
| 重连成功 | 重置计数器、恢复心跳 | — |
| 重连全部失败 | 显示 toast 警告 | `toast.warning("执行状态连接已断开")` |

### 3.4 心跳机制

- **间隔**：30 秒
- **客户端**发送 `{ "type": "ping" }`
- **服务端**回复 `{ "type": "pong" }`
- **前端**忽略 pong 消息（不触发 `onMessage` 回调）
- **连接断开**时清除心跳定时器

```ts
// 从开始节点提取输入参数定义
interface InputVariableDefinition {
  name: string;
  type: 'string' | 'number' | 'boolean' | 'json';
  description?: string;
  defaultValue?: any;
  required?: boolean;
}

// 动态渲染表单字段
const renderField = (variable: InputVariableDefinition) => {
  switch (variable.type) {
    case 'string':
      return <Input placeholder={variable.description || `输入 ${variable.name}`} />;
    case 'number':
      return <Input type="number" placeholder="0" />;
    case 'boolean':
      return <Switch />;
    case 'json':
      return (
        <div className="relative">
          <textarea
            className="code-input"
            placeholder='{"key": "value"}'
            style={{
              fontFamily: 'var(--font-mono)',
              fontSize: 'var(--text-sm)',
              minHeight: '80px',
              resize: 'vertical',
            }}
          />
          <span className="text-xs text-tertiary">JSON 格式</span>
        </div>
      );
  }
};
```

### 4.5 参数类型与组件映射

| 参数类型 | 表单组件 | 校验规则 | 默认值处理 |
|----------|---------|----------|------------|
| `string` | `<Input />` | 无特殊校验（可为空除非 required） | 显示 `defaultValue` |
| `number` | `<Input type="number" />` | 必须为有效数字 | 显示 `defaultValue` |
| `boolean` | `<Switch />` | — | `defaultValue ?? false` |
| `json` | `<textarea />`（monospace） | 提交时 `JSON.parse` 校验 | `JSON.stringify(defaultValue, null, 2)` |

### 4.6 校验规则

- **必填校验**：`required === true` 且值为空 → 字段下方显示红色提示 `"{变量名} 不能为空"`
- **类型校验**：
  - `number`：输入非数字 → `"请输入有效数字"`
  - `json`：`JSON.parse` 失败 → `"JSON 格式错误：{具体错误}"`
- **提交校验**：点击「开始执行」时遍历所有字段，有错误则阻止提交，第一个错误字段聚焦

### 4.7 弹窗交互细节

- 打开弹窗时，自动聚焦第一个输入字段
- `Enter` 键在 Input 中按下 → 不提交（避免误操作），仅在 textarea 中换行
- `Escape` 键 → 关闭弹窗
- 点击遮罩层 → 关闭弹窗
- 「开始执行」按钮在所有必填项校验通过前 `disabled`
- 工作流名称和版本号显示在表单顶部，帮助用户确认

```ts
  if (executionStatus !== 'running' && executionStatus !== 'waiting_review') return;

  const interval = setInterval(() => {
    setElapsed(Date.now() - new Date(startedAt).getTime());

  return () => clearInterval(interval);
}, [executionStatus, startedAt]);

// 格式化耗时
const formatDuration = (ms: number): string => {
  if (ms < 1000) return `${ms}ms`;
  if (ms < 60000) return `${(ms / 1000).toFixed(1)}s`;
  const minutes = Math.floor(ms / 60000);
  const seconds = Math.floor((ms % 60000) / 1000);
  return `${minutes}m ${seconds}s`;
};
```

```ts
// 当源节点完成且目标节点开始执行时，连线变为 active
const updateEdgeStates = (nodeStatuses: Map<string, NodeExecutionStatus>) => {
  workflowEditorStore.setEdges(
    edges.map((edge) => {
      const sourceStatus = nodeStatuses.get(edge.source);
      const targetStatus = nodeStatuses.get(edge.target);

      let edgeState: ExecutionEdgeState = 'idle';

      if (sourceStatus === 'success' && targetStatus === 'running') {
        edgeState = 'flowing';
      } else if (sourceStatus === 'success' && (targetStatus === 'success' || targetStatus === 'skipped')) {
        edgeState = 'active';
      }

      return {
        ...edge,
        className: `edge-${edgeState}`,
        animated: edgeState === 'flowing',
      };
    })
  );
};
```

```ts
// src/hooks/useExecutionMessageHandler.ts

export function useExecutionMessageHandler() {
  const store = useExecutionStore;
  const editorStore = useWorkflowEditorStore;

  const handleMessage = useCallback((message: WSServerMessage) => {
    switch (message.type) {
      case 'pong':
        // 忽略
        break;

      case 'execution_started':
        store.getState().setStatus('running');
        store.getState().setStartedAt(message.data.startedAt);
        store.getState().setExecutionId(message.data.executionId);
        // 初始化所有节点为 pending 状态
        const nodes = editorStore.getState().nodes;
        store.getState().initNodeStates(
          nodes.map((n) => ({ id: n.id, name: n.data.label, type: n.data.nodeType }))
        );
        break;

      case 'node_status':
        const { nodeId, status, input, output, meta, error } = message.data;
        store.getState().updateNodeStatus(nodeId, {
          status: status as NodeExecutionStatus,
          input: input || null,
          output: output || null,
          error: error || null,
          duration: meta?.duration || null,
          tokens: meta?.tokens ? {
            prompt_tokens: meta.tokens.prompt_tokens,
            completion_tokens: meta.tokens.completion_tokens,
            total_tokens: meta.tokens.total_tokens,
          } : null,
          estimatedCost: meta?.estimatedCost || null,
          model: meta?.model || null,
          agentName: meta?.agentName || null,
          startedAt: message.data.startedAt || null,
          completedAt: message.data.completedAt || null,
        });

        // 同步画布节点状态
        editorStore.getState().updateNodeData(nodeId, {
          executionStatus: status as any,
          executionResult: output || null,
          executionMeta: meta ? { duration: meta.duration, tokens: meta.tokens.total_tokens } : undefined,
        });

        // 更新统计
        if (status === 'success' || status === 'skipped') {
          store.getState().incrementCompletedNodes();
        }
        if (meta?.tokens) {
          store.getState().incrementTokens(meta.tokens);
        }
        if (meta?.estimatedCost) {
          store.getState().addCost(meta.estimatedCost);
        }
        break;

      case 'log':
        store.getState().addLog({
          id: `${message.data.nodeId}-${Date.now()}`,
          nodeId: message.data.nodeId,
          level: message.data.level,
          message: message.data.message,
          timestamp: message.data.timestamp,
        });
        break;

      case 'waiting_for_review':
        store.getState().setStatus('waiting_review');
        store.getState().setCurrentReviewNodeId(message.data.nodeId);
        store.getState().updateNodeStatus(message.data.nodeId, {
          status: 'waiting_review',
          input: message.data.inputData,
        });
        editorStore.getState().updateNodeData(message.data.nodeId, {
          executionStatus: 'waiting_review',
        });
        break;

      case 'review_timeout':
        store.getState().updateNodeStatus(message.data.nodeId, {
          status: 'error',
          error: '审核超时',
        });
        break;

      case 'execution_completed':
        if (message.data.status === 'completed') {
          store.getState().setCompleted(
            message.data.output,
            message.data.totalDuration
          );
        } else if (message.data.status === 'failed') {
          store.getState().setFailed(
            message.data.failedNodeId || '',
            message.data.errorMessage || 'Unknown error'
          );
        } else if (message.data.status === 'stopped') {
          store.getState().setStopped();
        }
        break;

      case 'execution_failed':
        store.getState().setFailed(
          message.data.failedNodeId,
          message.data.error.message
        );
        break;
    }
  }, []);

  return handleMessage;
}
```

## 13. 边界与异常

### 13.1 WebSocket 断连

| 场景 | 处理 |
|------|------|
| 网络波动导致断连 | 自动重连（指数退避，最多 5 次） |
| 重连中 | 面板顶部显示 "正在重连 (N/5)..." |
| 重连成功 | 从后端重新拉取当前执行状态（GET /api/executions/:id），同步所有节点状态 |
| 重连全部失败 | `toast.warning("执行状态连接已断开，请刷新页面查看最新状态")` |
| 断连期间后端状态变更 | 重连成功后通过 GET 接口补齐缺失的状态变更 |

### 13.2 执行超时

| 场景 | 处理 |
|------|------|
| 单个节点执行超时 | 后端返回 `node_status(error)` + error 信息包含 "timeout"，前端正常显示错误状态 |
| 审核节点超时 | 后端发送 `review_timeout` 消息，前端显示 toast + 节点变为 error |
| 整个执行超时 | 后端发送 `execution_failed`，前端显示失败视图 |
| 前端长时间无消息 | 心跳机制检测，如果 60s 内无任何消息（包括 pong），视为连接异常，主动断开并重连 |

### 13.3 并发执行

| 场景 | 处理 |
|------|------|
| 用户在同一 Tab 重复点击「运行」 | 执行中时「运行」按钮变为「停止」，无法重复启动 |
| 多个 Tab 打开同一工作流 | 各 Tab 独立执行，互不影响。后端管理并发 |
| 执行中刷新页面 | 执行继续在后端运行。刷新后从执行历史可找到该执行记录，但实时日志流会丢失（降级为查看历史日志） |

### 13.4 执行中编辑保护

| 场景 | 处理 |
|------|------|
| 执行中用户试图修改节点配置 | 右侧面板锁定为执行面板，无法选中节点打开属性面板 |
| 执行中用户试图删除节点 | 画布上的删除操作被禁用（`deleteKeyCode={null}` 动态切换） |
| 执行中用户试图拖拽新节点 | 节点库面板变灰 + 禁用拖拽 |
| 执行中用户试图连线 | 画布连线操作被禁用 |
| 编辑器顶部工具栏 | 「保存」「导入」「导出」按钮 disabled |

```ts
  s.status === 'running' || s.status === 'waiting_review'

<ReactFlow
  // ...

  nodesConnectable={!isExecuting}

```

### 13.5 大量日志性能

| 场景 | 处理 |
|------|------|
| 日志条数 > 1000 | 只渲染最近 500 条 + 顶部显示 "... 还有 N 条日志" |
| 日志增长过快 | 使用虚拟列表（可选，用 `react-window`） |
| 搜索时 | 搜索在完整日志数组上进行，不只是已渲染的 |

### 13.6 浏览器兼容性

| 场景 | 处理 |
|------|------|
| 浏览器不支持 WebSocket | 检测 `window.WebSocket`，不存在时降级为 HTTP 轮询（3 秒间隔 GET /api/executions/:id） |
| 浏览器不支持 CSS 动画 | 降级为静态颜色变化（不影响功能） |

### 13.7 错误消息处理

| 场景 | 处理 |
|------|------|
| API 返回 400（校验失败） | `toast.error(response.data.message)` |
| API 返回 404（执行不存在） | `toast.error("执行记录不存在")` + 跳转回编辑器 |
