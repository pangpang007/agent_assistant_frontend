# 汤圆代码助手 — Phase 2 前端开发文档：Agent 管理 + 工具系统 + 模型管理

> 本文档面向 Cursor / AI 编程助手，目标是根据本文档直接生成可运行的前端代码。
> 所有样式描述均引用 design token（CSS Variables），禁止硬编码色值。
> Phase 2 建立在 Phase 0（设计系统+脚手架）和 Phase 1（用户系统）完成的基础上，复用已有组件库、布局框架、路由体系、鉴权方案。

---

## 1. 目标

实现平台核心资源管理模块，包括三大功能域：

- **Agent 管理**：预置 6 个系统内置 Agent + 自定义 Agent 的 CRUD，完整的 Agent 配置表单
- **工具管理**：预置 7 个系统内置工具 + 自定义工具（通过 OpenAPI/Swagger 导入），含测试调用能力
- **模型管理**：多供应商配置（OpenAI/Anthropic/Google/自定义）、模型启用管理、API Key 脱敏、用量统计

#### 3.2.6 表单校验

- **必填字段**：名称、描述、角色描述、模型选择
- **校验时机**：
  - blur 时校验当前字段
  - submit 时校验所有字段
  - 已有错误的字段在输入时实时更新
- **校验失败时**：滚动到第一个错误字段位置（`element.scrollIntoView({ behavior: 'smooth', block: 'center' })`）

#### 3.2.7 提交逻辑

```
创建模式：
1. 校验通过
2. isSubmitting = true
3. 调用 POST /api/agents → 返回新建 Agent
4. 成功 → toast.success("Agent 创建成功") → navigate('/agents')
5. 失败 → toast.error(message) → isSubmitting = false

编辑模式：
1. 校验通过
2. isSubmitting = true
3. 调用 PUT /api/agents/:id → 返回更新后 Agent
4. 成功 → toast.success("Agent 更新成功") → navigate('/agents')
5. 失败 → toast.error(message) → isSubmitting = false
```

#### 3.2.8 编辑模式数据加载

- 进入编辑页时，调用 `GET /api/agents/:id` 获取 Agent 详情
- 加载中显示骨架屏（表单字段位置放置 Skeleton）
- 加载失败（如 Agent 不存在）→ `toast.error("Agent 不存在")` → `navigate('/agents', { replace: true })`

## 5. 状态管理

### 5.1 策略选择

Phase 2 **不新增全局 Zustand store**。理由：

- Agent 列表、工具列表、模型配置都是**页面级数据**，不需要跨页面共享
- 表单页的数据天然是局部 state
- 后续 Phase 3（工作流编辑器）如果需要引用 Agent 列表/模型列表，届时再提升为 store

### 5.2 数据获取模式

采用 **"service + useState + useEffect"** 模式（与 Phase 1 保持一致）：

```ts
// 典型列表页数据获取
const [agents, setAgents] = useState<Agent[]>([]);
const [isLoading, setIsLoading] = useState(true);

useEffect(() => {
  const fetchAgents = async () => {
    try {
      setIsLoading(true);
      const res = await agentService.getList({ type: typeFilter, search: searchQuery });
      setAgents(res.agents);
    } catch (error) {
      toast.error('加载 Agent 列表失败');
    } finally {
      setIsLoading(false);
    }
  };
  fetchAgents();
}, [typeFilter, searchQuery]);
```

### 5.3 需要共享的数据

以下数据可能需要跨组件共享，使用 **React Context** 或 **props 透传**：

- **Agent 配置页**：模型列表和工具列表需要从 modelService / toolService 获取，作为表单下拉的数据源。在 AgentFormPage 内部通过 useEffect 获取即可，无需提升。
- **模型管理页**：供应商列表和模型列表在页面组件内部 state 管理。
