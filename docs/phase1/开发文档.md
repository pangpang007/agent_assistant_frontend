## 1. 目标

实现完整的用户系统，包括：

- **注册**：个人注册（邮箱+密码+用户名）与团队注册（额外+团队名称），注册时选择类型
- **登录**：邮箱+密码登录，JWT token 管理
- **登出**：清除状态，跳转登录页
- **个人资料页**：修改用户名、头像（占位）、密码
- **团队管理**：加入团队（个人用户）/ 成员管理（团队 owner）
- **鉴权体系**：路由守卫、Token 自动刷新、角色判断
- **状态管理**：Zustand auth store 统一管理登录态

## 3. 鉴权方案

### 3.1 Token 存储

| localStorage Key | 值 | 说明 |
|------------------|---|------|
| `tangyuan_access_token` | JWT access_token 字符串 | 用于 API 请求鉴权，有效期 15 分钟 |
| `tangyuan_refresh_token` | JWT refresh_token 字符串 | 用于刷新 access_token，有效期 7 天 |
| `tangyuan_user` | JSON 序列化的用户信息 | 缓存用户基本信息，减少请求 |

> **命名规范**：统一前缀 `tangyuan_`，避免与其他应用冲突。

### 3.2 Axios 拦截器改造

在 `src/lib/axios.ts` 中改造拦截器，增加 Token 自动刷新机制。

#### 请求拦截器

```ts
instance.interceptors.request.use((config) => {
  const token = localStorage.getItem('tangyuan_access_token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
```

#### 响应拦截器（含 Token 自动刷新）

```ts
let isRefreshing = false;
let failedQueue: Array<{
  resolve: (value: any) => void;
  reject: (reason?: any) => void;
}> = [];

const processQueue = (error: any = null) => {
  failedQueue.forEach((prom) => {
    if (error) {
      prom.reject(error);
    } else {
      prom.resolve();
    }
  });
  failedQueue = [];
};

instance.interceptors.response.use(
  (response) => response.data,
  async (error) => {
    const originalRequest = error.config;

    // 401 且非刷新请求本身 → 尝试用 refresh_token 续期
    if (
      error.response?.status === 401 &&
      !originalRequest._retry
    ) {
      const refreshToken = localStorage.getItem('tangyuan_refresh_token');

      // 没有 refresh_token → 直接跳登录
      if (!refreshToken) {
        localStorage.removeItem('tangyuan_access_token');
        localStorage.removeItem('tangyuan_refresh_token');
        localStorage.removeItem('tangyuan_user');
        window.location.href = '/login';
        return Promise.reject(error);
      }

      // 已经在刷新中 → 排队等待
      if (isRefreshing) {
        return new Promise((resolve, reject) => {
          failedQueue.push({ resolve, reject });
        }).then(() => instance(originalRequest));
      }

      originalRequest._retry = true;
      isRefreshing = true;

      try {
        const { data } = await axios.post(
          `${instance.defaults.baseURL}/auth/refresh`,
          { refresh_token: refreshToken }
        );

        localStorage.setItem('tangyuan_access_token', data.access_token);
        localStorage.setItem('tangyuan_refresh_token', data.refresh_token);

        processQueue(null);
        return instance(originalRequest);
      } catch (refreshError) {
        processQueue(refreshError);
        // 刷新失败 → 清除所有 token，跳转登录
        localStorage.removeItem('tangyuan_access_token');
        localStorage.removeItem('tangyuan_refresh_token');
        localStorage.removeItem('tangyuan_user');
        window.location.href = '/login';
        return Promise.reject(refreshError);
      } finally {
        isRefreshing = false;
      }
    }

    // 非 401 错误 → Toast 提示
    const status = error.response?.status;
    const message = error.response?.data?.message;

    if (status === 403) {
      toast.error('无权限访问');
    } else if (status === 500 || status === 502 || status === 503) {
      toast.error('服务器异常，请稍后重试');
    } else if (!error.response) {
      toast.error('网络连接失败，请检查网络');
    } else if (message) {
      // 后端返回的自定义错误消息（如"邮箱已注册"等），交由调用方处理
      // 这里不做全局 toast，由业务代码自行决定
    }

    return Promise.reject(error);
  }
);
```

#### 关键设计

- **并发请求处理**：多个请求同时 401 时，只发一次刷新请求，其他请求排队等待
- **静默刷新**：用户在正常使用时 token 过期，自动续期，用户无感知
- **刷新失败处理**：清除所有 token，强制跳转登录页

### 3.3 AuthGuard 路由守卫组件

**文件路径**：`src/components/auth/AuthGuard.tsx`

#### 实现逻辑

```ts
interface AuthGuardProps {
  children: React.ReactNode;
}
```

组件内部：

1. 从 `useAuthStore` 获取 `isAuthenticated` 和 `isLoading`
2. 若 `isLoading === true`（正在初始化/获取用户信息）→ 渲染全屏 Spinner
3. 若 `isAuthenticated === false` → `<Navigate to="/login" replace />` 重定向
4. 若 `isAuthenticated === true` → 渲染 `{children}`

#### 页面加载时初始化

在 `AuthGuard` 的 `useEffect` 中（或在更上层的 App 初始化逻辑中）：
- 如果 `localStorage` 中有 `tangyuan_access_token`，但 store 中 `user` 为空 → 调用 `GET /api/user/profile` 获取用户信息并填充 store
- 如果请求失败（token 无效/过期）→ 清除 token，标记未登录

#### 在路由中使用

```tsx
// 在 AppLayout 内部包裹需要鉴权的路由
<Route element={<AuthGuard><AppLayout /></AuthGuard>} >
  <Route index element={<DashboardPage />} />
  {/* ...其他需鉴权路由 */}
</Route>
```

> **替代方案**：也可以在 `AppLayout` 组件内部第一行调用 `AuthGuard` 逻辑，效果相同。

### 3.4 反向守卫（已登录不访问登录/注册页）


### 3.5 角色判断逻辑

#### 用户类型（user_type）

| 值 | 含义 | 权限 |
|----|------|------|
| `personal` | 个人用户 | 无团队功能，可加入团队 |
| `team_owner` | 团队创建者 | 管理团队成员、查看团队邀请码 |
| `team_member` | 团队成员 | 查看团队信息 |

#### 判断逻辑

```ts
// auth store 中
type UserRole = 'personal' | 'team_owner' | 'team_member';

interface User {
  id: string;
  email: string;
  username: string;
  avatar_url: string | null;
  user_type: 'personal' | 'team';  // 注册时选择的类型
  role: UserRole;                   // 当前角色
  team_id: string | null;           // 所属团队 ID（null 表示无团队）
  team_name: string | null;         // 所属团队名称
}
```

- 个人注册用户，未加入团队：`user_type = 'personal'`, `role = 'personal'`, `team_id = null`
- 个人注册用户，加入团队后：`role = 'team_member'`, `team_id = <团队ID>`
- 团队注册：`user_type = 'team'`, `role = 'team_owner'`, `team_id = <团队ID>`

#### 4.1.7 提交逻辑

```
1. 前端校验 → 失败则设置 errors，终止
2. isSubmitting = true
3. 调用 authStore.login({ email, password })
4. 成功 → toast.success("登录成功") → navigate("/", { replace: true })
5. 失败 → 根据错误码处理：
   - 401 → setErrors({ password: "邮箱或密码错误" })
   - 网络错误 → toast.error("网络连接失败，请检查网络")
   - 其他 → toast.error(message || "登录失败，请重试")
6. isSubmitting = false
```

- **输入时**：如果该字段已有错误，输入时实时更新校验（去除错误提示）

#### 方法列表

```ts
const authService = {
  /**
   * 用户登录
   * POST /api/auth/login
   */
  login: (data: LoginRequest): Promise<LoginResponse> =>
    http.post('/auth/login', data),

  /**
   * 用户注册
   * POST /api/auth/register
   */
  register: (data: RegisterRequest): Promise<RegisterResponse> =>
    http.post('/auth/register', data),

  /**
   * 刷新 Token
   * POST /api/auth/refresh
   * 注意：此请求不经过拦截器（避免循环），直接使用 axios
   */
  refreshToken: (data: RefreshTokenRequest): Promise<RefreshTokenResponse> =>
    axios.post(`${BASE_URL}/auth/refresh`, data),

  /**
   * 登出
   * POST /api/auth/logout
   */
  logout: (): Promise<void> =>
    http.post('/auth/logout'),
};
```

## 6. 状态管理

### 6.1 Auth Store（Zustand）

**文件路径**：`src/stores/authStore.ts`

#### 完整定义

```ts
interface AuthState {
  // --- 状态 ---
  user: UserInfo | null;
  accessToken: string | null;
  refreshToken: string | null;
  isAuthenticated: boolean;
  isLoading: boolean;              // 初始化加载中（检查 token、获取用户信息）

  // --- 方法 ---
  /** 初始化：从 localStorage 恢复状态，验证 token 有效性 */
  initialize: () => Promise<void>;

  /** 登录 */
  login: (email: string, password: string) => Promise<void>;

  /** 注册 */
  register: (data: RegisterRequest) => Promise<void>;

  /** 登出：调用 API + 清除状态 + 跳转 */
  logout: () => Promise<void>;

  /** 更新用户信息（如修改个人资料后） */
  updateUser: (user: Partial<UserInfo>) => void;

  /** 清除所有认证状态 */
  clearAuth: () => void;
}
```

#### 实现要点

```ts
export const useAuthStore = create<AuthState>((set, get) => ({
  user: null,
  accessToken: null,
  refreshToken: null,
  isAuthenticated: false,
  isLoading: true,                // 初始为 true，initialize 完成后设为 false

  initialize: async () => {
    try {
      const accessToken = localStorage.getItem('tangyuan_access_token');
      const refreshToken = localStorage.getItem('tangyuan_refresh_token');
      const cachedUser = localStorage.getItem('tangyuan_user');

      if (!accessToken || !refreshToken) {
        set({ isLoading: false });
        return;
      }

      // 尝试从缓存恢复
      if (cachedUser) {
        const user = JSON.parse(cachedUser);
        set({ user, accessToken, refreshToken, isAuthenticated: true, isLoading: false });
      } else {
        set({ accessToken, refreshToken });
      }

      // 从服务器获取最新用户信息
      const profile = await userService.getProfile();
      localStorage.setItem('tangyuan_user', JSON.stringify(profile));
      set({ user: profile, isAuthenticated: true, isLoading: false });
    } catch {
      // token 无效 → 清除
      get().clearAuth();
    }
  },

  login: async (email, password) => {
    const response = await authService.login({ email, password });
    const { access_token, refresh_token, user } = response;

    localStorage.setItem('tangyuan_access_token', access_token);
    localStorage.setItem('tangyuan_refresh_token', refresh_token);
    localStorage.setItem('tangyuan_user', JSON.stringify(user));

    set({
      user,
      accessToken: access_token,
      refreshToken: refresh_token,
      isAuthenticated: true,
    });
  },

  register: async (data) => {
    await authService.register(data);
    // 注册后不自动登录，引导用户去登录页
  },

  logout: async () => {
    try {
      await authService.logout();
    } catch {
      // 即使 API 失败也清除本地状态
    }
    get().clearAuth();
  },

  updateUser: (partial) => {
    const current = get().user;
    if (!current) return;
    const updated = { ...current, ...partial };
    set({ user: updated });
    localStorage.setItem('tangyuan_user', JSON.stringify(updated));
  },

  clearAuth: () => {
    localStorage.removeItem('tangyuan_access_token');
    localStorage.removeItem('tangyuan_refresh_token');
    localStorage.removeItem('tangyuan_user');
    set({
      user: null,
      accessToken: null,
      refreshToken: null,
      isAuthenticated: false,
      isLoading: false,
    });
  },
}));
```

#### 在 App 入口使用

```tsx
// App.tsx 或 main.tsx 中
function App() {
  const { initialize, isLoading } = useAuthStore();

  useEffect(() => {
    initialize();
  }, []);

  if (isLoading) {
    return (
      <div className="min-h-screen flex items-center justify-center" style={{ background: 'var(--bg-canvas)' }}>
        <Spinner size="lg" />
      </div>
    );
  }

  return <RouterProvider router={router} />;
}
```

## 9. 交互细节

### 9.1 表单校验时机

| 时机 | 行为 |
|------|------|
| **blur** | 字段失去焦时校验该字段，显示错误 |
| **submit** | 提交时校验所有字段，即使未 blur 过 |
| **input（已有错误时）** | 如果该字段已显示错误，输入时实时重新校验，通过则立即移除错误提示 |
| **input（无错误时）** | 不主动校验，避免干扰用户输入 |

### 9.2 密码规则

| 规则 | 说明 |
|------|------|
| 最小长度 | 8 个字符 |
| 大写字母 | 至少 1 个 A-Z |
| 小写字母 | 至少 1 个 a-z |
| 数字 | 至少 1 个 0-9 |
| 特殊字符 | 可选，但有的话增强强度 |

**校验正则**：`/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/`

## 10. 边界与异常

### 10.1 邮箱已注册

- **触发**：注册时后端返回 `409 Conflict`
- **前端处理**：
  1. 在邮箱字段下方显示错误：`"该邮箱已注册，请直接登录"`，`color: var(--accent-danger)`
  2. 在错误提示下方显示链接：`<Link to="/login">去登录 →</Link>`，`color: var(--accent-primary)`
  3. 不弹 toast（避免干扰）

### 10.2 团队邀请码无效

- **触发**：加入团队时后端返回 `404` 或 `400`（邀请码不存在/已过期/已失效）
- **前端处理**：
  1. 在邀请码输入框下方显示错误：`"邀请码不存在或已过期"`
  2. 同时 `toast.error("邀请码不存在或已过期")`

### 10.3 Token 过期

- **触发**：API 请求返回 `401 Unauthorized`
- **前端处理**：
  1. Axios 拦截器自动用 `refresh_token` 请求新的 `access_token`
  2. 刷新成功 → 自动重试原请求，用户无感知
  3. 刷新失败（refresh_token 也过期）→ 清除所有 token → 跳转 `/login`
  4. 用户可能需要重新登录，显示 toast：`"登录已过期，请重新登录"`

### 10.4 网络错误

- **触发**：请求无响应（`!error.response`）
- **前端处理**：`toast.error("网络连接失败，请检查网络")`
- **按钮状态**：`isSubmitting` 设为 `false`，按钮恢复可点击

### 10.5 团队 owner 删除账户

- **后端逻辑**：删除 owner 账户时自动删除该团队
- **前端处理**：在个人资料页"删除账户"危险区域的描述中明确提示：`"如果你是团队 owner，团队也将被删除。"`
- 删除成功后：清除所有状态 → 跳转 `/login` → `toast.success("账户已删除")`

### 10.6 移除最后一个成员（非 owner）

- 不涉及（owner 不能移除自己，owner 只能删除账户来离开团队）

### 10.7 并发操作

- 同一页面同时操作（如两个 owner 同时移除成员）：后端通过事务保证一致性，前端在收到 404/409 时刷新页面数据并 toast 提示 `"数据已变更，请刷新页面"`

### 10.8 空状态处理

- 成员列表为空（极端情况，至少应有 owner）：不展示列表，显示 EmptyState 组件
  - 图标：`Users`
  - 标题："暂无成员"
  - 描述："邀请团队成员开始协作"
