# 汤圆代码助手 — Phase 3 前端开发文档：知识库管理（RAG）

> 本文档面向 Cursor / AI 编程助手，目标是根据本文档直接生成可运行的前端代码。
> 所有样式描述均引用 design token（CSS Variables），禁止硬编码色值。
> Phase 3 建立在 Phase 0（设计系统+脚手架）、Phase 1（用户系统）、Phase 2（Agent+工具+模型管理）完成的基础上，复用已有组件库、布局框架、路由体系、鉴权方案、service 层模式。

---

## 1. 目标

实现知识库管理（RAG）模块，包括：

- **知识库列表页**：卡片式展示所有知识库，支持创建/删除操作
- **知识库详情页**：文档管理（上传/列表/状态/删除）+ 分块配置 + 检索测试
- **文件上传系统**：拖拽上传 + 点击上传，支持多文件批量上传，含进度条、格式校验、错误处理
- **检索测试面板**：输入查询词，展示 Top-K 检索结果（含相似度分数、来源文件、文本高亮）

#### 3.3.4 上传进度轮询机制

```ts
// 上传流程伪代码
async function handleUpload(files: File[]) {
  for (const file of files) {
    // 1. 前端校验
    const validationError = validateFile(file);
    if (validationError) {
      updateQueueItem(file.id, { status: 'error', error: validationError });
      continue;
    }
    
    // 2. 上传文件（使用 XMLHttpRequest 或 axios onUploadProgress 获取上传进度）
    updateQueueItem(file.id, { status: 'uploading', progress: 0 });
    
    try {
      await knowledgeService.uploadDocument(kbId, file, (progress) => {
        updateQueueItem(file.id, { status: 'uploading', progress });
      });
      
      // 3. 上传完成，开始轮询文档处理状态
      updateQueueItem(file.id, { status: 'processing' });
      pollDocumentStatus(kbId, file.documentId);
      
    } catch (error) {
      updateQueueItem(file.id, { status: 'error', error: '上传失败' });
    }
  }
}

// 轮询文档处理状态（每 2 秒一次，最多轮询 60 次 = 2 分钟）
async function pollDocumentStatus(kbId: string, docId: string) {
  const maxRetries = 60;
  let retries = 0;
  
  const poll = async () => {
    if (retries >= maxRetries) {
      updateQueueItem(docId, { status: 'error', error: '处理超时' });
      return;
    }
    
    const doc = await knowledgeService.getDocument(kbId, docId);
    if (doc.status === 'ready') {
      updateQueueItem(docId, { status: 'success' });
      refreshDocumentList();
      return;
    }
    if (doc.status === 'failed') {
      updateQueueItem(docId, { status: 'error', error: '处理失败' });
      refreshDocumentList();
      return;
    }
    
    retries++;
    setTimeout(poll, 2000);
  };
  
  poll();
}
```

- **上传进度**：使用 `axios` 的 `onUploadProgress` 回调获取实时上传进度
- **处理状态轮询**：上传完成后，文档需要后端处理（文本提取→分块→Embedding→存储），此过程通过轮询 `GET /api/knowledge-bases/:id/documents/:docId` 获取状态
- **轮询间隔**：2 秒
- **最大轮询次数**：60 次（2 分钟后显示超时）
- **所有文件上传完成后**：自动刷新文档列表

### 4.2 knowledgeService 方法

```ts
import http from '@/lib/axios';

const knowledgeService = {
  // ===== 知识库 CRUD =====

  /**
   * 获取知识库列表
   * GET /api/knowledge-bases
   */
  getList: (): Promise<KnowledgeBaseListResponse> =>
    http.get('/knowledge-bases'),

  /**
   * 获取单个知识库详情
   * GET /api/knowledge-bases/:id
   */
  getById: (id: string): Promise<KnowledgeBase> =>
    http.get(`/knowledge-bases/${id}`),

  /**
   * 创建知识库
   * POST /api/knowledge-bases
   */
  create: (data: CreateKnowledgeBaseRequest): Promise<KnowledgeBase> =>
    http.post('/knowledge-bases', data),

  /**
   * 更新知识库（名称、描述、分块配置）
   * PUT /api/knowledge-bases/:id
   */
  update: (id: string, data: UpdateKnowledgeBaseRequest): Promise<KnowledgeBase> =>
    http.put(`/knowledge-bases/${id}`, data),

  /**
   * 删除知识库
   * DELETE /api/knowledge-bases/:id
   */
  delete: (id: string): Promise<void> =>
    http.delete(`/knowledge-bases/${id}`),

  // ===== 文档管理 =====

  /**
   * 获取文档列表
   * GET /api/knowledge-bases/:id/documents
   */
  getDocuments: (kbId: string): Promise<DocumentListResponse> =>
    http.get(`/knowledge-bases/${kbId}/documents`),

  /**
   * 获取单个文档详情
   * GET /api/knowledge-bases/:id/documents/:docId
   */
  getDocument: (kbId: string, docId: string): Promise<KnowledgeDocument> =>
    http.get(`/knowledge-bases/${kbId}/documents/${docId}`),

  /**
   * 上传文档
   * POST /api/knowledge-bases/:id/documents
   * Content-Type: multipart/form-data
   */
  uploadDocument: (
    kbId: string,
    file: File,
    onProgress?: (progress: number) => void,
  ): Promise<KnowledgeDocument> =>
    http.post(
      `/knowledge-bases/${kbId}/documents`,
      (() => {
        const formData = new FormData();
        formData.append('file', file);
        return formData;
      })(),
      {
        headers: { 'Content-Type': 'multipart/form-data' },
        onUploadProgress: (progressEvent) => {
          if (progressEvent.total && onProgress) {
            const percentCompleted = Math.round(
              (progressEvent.loaded * 100) / progressEvent.total
            );
            onProgress(percentCompleted);
          }
        },
      },
    ),

  /**
   * 删除文档
   * DELETE /api/knowledge-bases/:id/documents/:docId
   */
  deleteDocument: (kbId: string, docId: string): Promise<void> =>
    http.delete(`/knowledge-bases/${kbId}/documents/${docId}`),

  /**
   * 重新处理文档（失败时重试）
   * POST /api/knowledge-bases/:id/documents/:docId/reprocess
   */
  reprocessDocument: (kbId: string, docId: string): Promise<KnowledgeDocument> =>
    http.post(`/knowledge-bases/${kbId}/documents/${docId}/reprocess`),

  // ===== 检索测试 =====

  /**
   * 检索测试
   * POST /api/knowledge-bases/:id/search
   */
  search: (kbId: string, data: RetrievalQuery): Promise<RetrievalResponse> =>
    http.post(`/knowledge-bases/${kbId}/search`, data),
};

export default knowledgeService;
```

## 7. 交互细节

### 7.1 上传进度轮询机制

详见 3.3.6 节。核心逻辑：

1. 文件上传使用 `axios` 的 `onUploadProgress` 获取上传进度（0-100%）
2. 上传完成后，后端返回 `document_id`，进入处理状态轮询
3. 轮询 `GET /api/knowledge-bases/:id/documents/:docId`，每 2 秒一次
4. 状态变为 `ready` 或 `failed` 时停止轮询
5. 最长轮询 2 分钟（60 次），超时标记为"处理超时"
6. 文档列表区域的轮询（3 秒间隔）独立于上传队列轮询，用于刷新列表中所有处理中文档的状态

### 7.2 文档状态实时更新

- 详情页通过 `useEffect` 监听文档列表中是否有 `processing` 状态的文档
- 如果有，启动 3 秒间隔的定时器，调用 `GET /api/knowledge-bases/:id/documents` 刷新整个列表
- 当所有文档不再是 `processing` 状态时，自动停止定时器
- 组件 unmount 时清理所有定时器（`clearInterval`）
- 后续可升级为 WebSocket 推送，但目前轮询方案足够

### 7.3 检索结果的展开/收起

- `RetrievalResultCard` 组件内部管理 `isExpanded` 状态
- 初始状态：`max-height: 160px; overflow: hidden;` + 底部渐变遮罩
- 点击「展开」：`max-height: none;` 移除遮罩，按钮变为「收起」
- 点击「收起」：恢复折叠状态
- 切换动画：`transition: max-height var(--duration-slow) var(--ease-default);`

### 7.4 分块配置的实时预览

- 滑块拖动时，当前值实时更新（无需 debounce）
- 保存按钮的启用/禁用通过对比当前值与 API 返回的原始值判断
- 如果有修改但未保存，离开页面时提示（使用 `useBlocker`）：
  - "分块配置已修改，是否放弃保存？"
  - [继续编辑] [放弃并离开]

### 7.5 知识库列表卡片进入动画

- 与 Phase 2 Agent 卡片一致的动画方案
- `@keyframes fadeInUp`，每个卡片延迟 `50ms * index`

### 7.6 删除知识库后的处理

- 如果是在详情页删除：`navigate('/knowledge', { replace: true })`

## 8. 边界与异常

### 8.1 上传超大文件（> 50MB）

- **前端校验**：文件选择后立即检查 `file.size > 50 * 1024 * 1024`
- **处理**：文件加入上传队列，状态标记为 `error`，错误信息："文件大小超过 50MB 限制"
- **不执行上传**，用户可以手动从队列中移除

### 8.2 文件格式不支持

- **前端校验**：根据文件扩展名判断
- **处理**：文件加入上传队列，状态标记为 `error`，错误信息："不支持的文件格式，仅支持 PDF、TXT、Markdown、CSV、DOCX"
- **Toast 提示**：如果批量选择中有不支持的文件，`toast.warning("{N} 个文件格式不支持，已跳过")`

### 8.3 文档处理失败的重试逻辑

- **状态展示**：`DocumentStatusBadge` 显示红色"失败"标记
- **操作按钮**：显示「重试」按钮
- **重试流程**：
  1. 点击「重试」→ 按钮变为 loading 态
  2. 调用 `POST /api/knowledge-bases/:id/documents/:docId/reprocess`
  3. 成功：文档状态变为 `processing`，`DocumentStatusBadge` 更新为蓝色"处理中"
  4. 失败：`toast.error("重试失败，请检查文件后重新上传")`
- **重试后**：重新进入轮询逻辑（3 秒间隔刷新文档列表）

### 8.4 检索无结果

- 详见 3.5.5 节
- 显示空状态图标 + 文案

### 8.5 知识库为空时的提示

- 知识库列表页空状态见 3.1.9 节
- 详情页文档列表空状态见 3.3.5 节

### 8.6 知识库详情页 404

- `GET /api/knowledge-bases/:id` 返回 404
- 处理：`toast.error("知识库不存在或已被删除")` → `navigate('/knowledge', { replace: true })`

### 8.7 上传中断网

- 正在上传的文件：`axios` 抛出网络错误
- 处理：文件状态标记为 `error`，错误信息："网络连接失败"
- 提示：`toast.error("网络断开，上传中断")`
- 用户可恢复网络后点击「重试」（重新加入上传队列）

### 8.8 并发删除冲突

- 用户 A 在列表页删除了知识库 X，同时用户 B 在知识库 X 的详情页

export function formatFileSize(bytes: number): string {
  if (bytes === 0) return '0 B';
  const units = ['B', 'KB', 'MB', 'GB'];
  const i = Math.floor(Math.log(bytes) / Math.log(1024));
  const size = (bytes / Math.pow(1024, i)).toFixed(i > 0 ? 1 : 0);
  return `${size} ${units[i]}`;
}

export function formatRelativeTime(dateStr: string): string {
  const now = new Date();
  const date = new Date(dateStr);
  const diffMs = now.getTime() - date.getTime();
  const diffMin = Math.floor(diffMs / 60000);
  const diffHour = Math.floor(diffMs / 3600000);
  const diffDay = Math.floor(diffMs / 86400000);

  if (diffMin < 1) return '刚刚';
  if (diffMin < 60) return `${diffMin} 分钟前`;
  if (diffHour < 24) return `${diffHour} 小时前`;
  if (diffDay < 30) return `${diffDay} 天前`;
  return formatDate(dateStr);
}
